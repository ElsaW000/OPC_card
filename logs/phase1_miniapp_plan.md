# OPC名片 一期功能排期表

> 更新时间：2026-03-26
> 阶段目标：先完成微信小程序 MVP，上线验证“创建名片 -> 发出名片 -> 形成连接”的核心闭环
> 后续方向：在小程序验证后，再规划 App 承接关系维护、通知、深度 AI 与更完整的数据能力

## 一期范围定义

一期只做小程序，不追求完整社交网络，不追求所有 AI 能力，不追求复杂数据分析。

一期核心目标：

1. 用户能创建并保存自己的名片
2. 用户能展示、分享、交换名片
3. 系统能沉淀基础联系人或交换记录
4. 数据和云端能力可真实运行，不依赖纯静态 mock

## P0 - 必须完成

### 1. 云环境与真实数据接通
- 内容：
  - 配置微信云开发环境 ID
  - 创建数据库集合
  - 上传并部署云函数
  - 验证编辑/读取链路真实可用
- 当前状态：进行中
- 说明：
  - 代码已接入 `wx.cloud.init`
  - `app.js` 中环境 ID 仍是占位值

### 2. 单用户名片主链路跑通
- 内容：
  - 创建/编辑名片
  - 保存名片
  - 读取并展示名片详情
  - 默认名片展示稳定
- 当前状态：半完成
- 已有：
  - `edit -> saveCard`
  - `cardDetail -> getCard`
- 缺口：
  - 需要真云环境验证
  - 字段结构仍需统一

### 3. 多身份名片基础版
- 内容：
  - 支持多张名片
  - 设置默认名片
  - 在“我的名片”页展示真实列表
- 当前状态：未完成
- 已有：
  - 页面有多身份视觉稿
  - 有 `getCards` / `setDefaultCard`
- 缺口：
  - `mycards` 仍是本地 mock
  - `saveCard` 仍偏单卡模型

### 4. 交换名片基础闭环
- 内容：
  - 从默认名片进入交换页
  - 扫码/二维码/分享形成交换入口
  - 至少记录一次交换行为
- 当前状态：半完成
- 已有：
  - 交换页 UI 和读取默认名片
  - 二维码页入口
- 缺口：
  - 扫码后未形成真实交换记录
  - 联系人沉淀未打通

### 5. 联系人或交换记录二选一先做真链路
- 内容：
  - 至少真实落地一个集合：
    - `contacts`
    - 或 `exchange_records`
  - 页面不再完全依赖 mock 数据
- 当前状态：未开始
- 说明：
  - 一期建议优先做 `exchange_records`，再映射出联系人列表

### 6. 分享与传播入口可演示
- 内容：
  - 小程序分享
  - 名片二维码
  - 名片详情页可稳定打开
- 当前状态：半完成
- 已有：
  - 分享相关按钮和分享方法
  - 二维码页面已注册
- 缺口：
  - 传播后的打开路径和参数还没完整收口

## 推荐执行顺序

1. 先接通真云环境
2. 跑通单用户名片保存/读取

4. 打通交换记录或联系人沉淀
5. 收口分享/二维码打开路径
6. 最后再补 AI、访客、管理页增强

## 一期验收标准

满足以下条件，可认为小程序一期基本完成：

1. 真实用户可以创建并保存名片
2. 默认名片、至少两张身份名片可以管理
3. 名片可被分享、打开、交换
4. 至少沉淀一种真实连接数据（联系人或交换记录）
5. 所有主页面不再依赖关键 mock 数据
6. 云函数、数据库、AI 基础能力在真实环境下可运行
