## 一、产品定位

帮助 AI 时代的新个体，更轻、更快地介绍自己、建立连接、沉淀联系人。

**它不是传统销售型商务名片，而是更适合展示"我是谁、我在做什么、我能和你如何连接"的个人连接工具。**

---

### 产品策略
- **微信小程序**：获客渠道，用于快速传播和获取新用户
- **APP**：最终产品，用户注册后会导流到APP使用完整功能

---

## 二、目标用户

面向 AI 时代快速涌现的新个体与轻团队：
- OPC / 一人公司
- 独立开发者
- 小公司或微型团队创始人
- 独立游戏制作人
- 漫剧作者 / 创作者
- 设计师 / 插画师 / 数字内容创作者
- 项目制自由职业者

---

## 三、字段设计

### 按功能分类

#### 1. 基础信息
- 姓名
- 头像
- 身份标签
- 一句话介绍
- 学校/学历（可选：学信网绑定验证 - 后期功能，需要API对接）

#### 2. 联系方式（可设置公开/非公开）
- 电话
- 邮箱
- 微信
- 其他联系方式

#### 3. 职业信息
- 职位
- 公司
- 当前项目
- 工作年限

#### 4. 作品/产品展示
- 作品链接
- 项目展示
- 视频

#### 5. 社交媒体
- GitHub
- Twitter/X
- 抖音/B站
- 其他社媒入口

#### 6. 隐私设置（安全性）
- **公开**：拿到名片的人都能看到
- **非公开**：只有交换成功后才能看到
- 每个字段可单独设置

## 五、页面跳转关系

### 5.1 我的名片 (mycards) - TabBar 首页
- 点击卡片 → 预览名片详情（cardDetail，owner 视图：有编辑/二维码按钮）
- 点击"编辑" → 跳转 edit 页面
- 点击二维码图标 → 跳转 qrcode 页面（也可从 cardDetail 进入）
- 点击"设为默认" → 更新默认名片
- 点击"+" → 创建新身份
- 底部"交换名片"按钮 → 跳转 exchange 页面

### 5.2 联系人 (contacts) - TabBar
- 这是**交换过名片**的联系人列表
- **待处理请求** - 顶部显示有人要跟你交换名片
  - 点同意 → 交换成功，双方成为联系人
  - 点拒绝 → 拒绝交换
- **AI助手提示**：顶部显示"谁谁谁更新了产品哦，可以去看看"（如果有更新）
- 点击联系人 → 查看联系人名片详情（cardDetail，visitor 模式）
- 点击"电话" → 弹出确认拨打对话框 (wx.makePhoneCall)
- 点击"备忘录" → 添加/查看联系人备忘录
- （"聊天"按钮暂不实现，如无法跳转微信则去掉）

### 5.3 工作台 (home) - TabBar

**a. AI社交助理**
- 对话框形式，和 AI 功能页面 (aiFeatures) 是同一个
- 客户手动输入一句话 → 跳转到 AI 功能页面继续对话
- 默认招呼语："您有什么需求？"、"看下最近圈子里有什么新鲜事？"
- AI助手分析联系人和访客，可选模型（免费额度有限，部分付费）

**b. 数据模块**
- 访客数据：每周总访问数字
- 访客画像：气泡图/云图（按名片编辑时的AI识别标签分类，气泡大小=数量）
- 点击箭头 → 跳转到数据分析页面 (analytics)

**c. 星标联系人**
- 跳转联系人页面，筛选展示星标联系人
- 点击联系人 → 查看名片详情
- 当联系人更新产品/视频时，显示红点提示

**d. 系统设置**
- AI语气设置
- 是否公开名片动态
- 隐私设置
- **黑名单** - 屏蔽不想收到交换请求的人

**e. 底部**
- 会员权益介绍
- 联系我们 → 弹出表单收集建议

### 5.4 管理 (management) - TabBar

个人账号管理，当前功能：
- 跳转名片二维码页（qrcode）
- 跳转会员权益页（member）

> 后期扩展：名片模板管理、账号设置、数据导出

### 5.5 交换名片 (exchange)

双 Tab 设计：

**Tab 1 — 我要递出**
- 展示当前默认名片预览（Banner + 头像 + 姓名/职位/公司）
- 二维码展示区（供对方扫码）
- 操作按钮：
  - `发给微信好友`（open-type="share"）
  - `保存二维码`
  - `换一张`（切换其他名片）

**Tab 2 — 我要接收**
- 说明文字（不显示扫码框）
- `开始扫描` → 调起微信扫一扫
- 输入框 + `输入交换码` → 手动输入交换码

### 5.6 名片详情 (cardDetail) - 两种视图

**Owner 视图**（从 mycards 进入，非 visitor 模式）
- 完整展示名片内容
- 顶部操作栏：设为默认 / 二维码图标 / 编辑按钮

**Visitor 视图**（从扫码/分享/contacts 进入，携带 `visitor=1`）
- 完整展示名片内容（按隐私设置过滤非公开字段）
- 底部固定按钮：`交换名片`

### 5.7 名片二维码 (qrcode)

- 展示当前名片的二维码（目前为模拟格子图，后期对接真实 wxacode）
- 名片缩略信息：头像 + 姓名 + 职位
- 操作：`分享名片`（onShareAppMessage）/ `复制页面路径`

### 5.8 访客记录 (visitor)
- 展示访客列表（谁扫了我的二维码）
- 免费用户：只显示前3个基础信息（无联系方式）
- 付费用户：显示完整信息+名片详情

### 5.9 数据分析 (analytics)
- 访客数据详情
- 交换数据统计
- AI优化建议

### 5.10 AI功能 (aiFeatures)
- AI聊天助手对话框，可选模型（免费额度有限，部分付费）
- 来访者分析
- 联系人推荐
- 话术建议

## 六、核心功能设计

### 6.1 隐私功能 - 公开/非公开设置
每个字段可独立设置：公开（所有人可见）/ 非公开（仅交换成功后可见）

### 6.2 快速交换名片流程
1. 递出方：exchange → 我要递出 → 展示二维码
2. 接收方：exchange → 我要接收 → 扫码 or 输入交换码
3. 接收方扫码后 → 进入对方 cardDetail（visitor 模式）→ 点击"交换名片"
4. 递出方收到请求 → contacts 顶部待处理 → 同意/拒绝

### 6.3 访客画像分析
气泡图/云图，Canvas 绘制，气泡位置随机分布，大小按数量比例

### 6.4 AI公关助手
分析联系人和访客，提供话术建议、联系人推荐、跟进任务

### 6.5 云数据同步
微信云开发，名片数据、访客记录、联系人关系云端存储

### 6.6 联系人动态推送
联系人更新名片/产品/视频时，在 contacts 和工作台星标区显示提示

## 七、技术说明

1. **跳转微信聊天**：暂时不做（"聊天"按钮可能去掉）
2. **分享**：微信内分享使用 `open-type="share"`，暂不支持分享到小红书
3. **电话拨打**：`wx.makePhoneCall`
4. **气泡图**：Canvas 绘制
5. **二维码**：当前为模拟格子图，后期对接 `wx.cloud.callFunction` 生成真实 wxacode
