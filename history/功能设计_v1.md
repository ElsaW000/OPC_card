# OPC 名片 - 功能设计文档

## （Phase 2）关系网功能
> 依赖联系人数据积累，≥10人后开放

### 1.1 功能描述
像Obsidian一样的关系图谱，AI帮用户整理联系人。

### 1.2 展示方式
- 每个人有自己的关系网页面
- 按tag划分颜色（职业、行业、title）
- 隐私优先：不默认公开

### 1.3 隐私控制
- 默认私有：自己可见
- 可选择开放给特定人
- 不公开"谁认识谁"

### 1.4 Tag分类
- 职业（程序员、设计师、运营...）
- 行业（AI、金融、教育...）
- Title（CEO、VP、Founder...）
- 自定义标签

## 2. 社交 - 碰一碰功能

### 2.1 核心流程
1. 用户A碰用户B的NFC标签
2. 打开小程序并进入交换确认页/弹层
3. 用户确认是否交换名片
4. 交换成功后，双方在"名片簿"中可查看对方

### 2.2 交换后展示
- 交换记录持久保存
- 划掉小程序也能找回
- 头像+名字+基本信息

### 2.3 微信扩展
- 碰完可以问"是否加微信"
- 加微信需要对方同意

### 2.X 远程发名片
- 用户可在微信聊天中转发自己的小程序名片
- 对方点击后可查看并保存
- 支持回传名片（形成交换记录）

### 2.4 技术实现

#### 方案A：NFC标签 + 小程序按钮（推荐）
- 用户贴一个NFC标签（成本几块钱）
- 碰一下 → 打开小程序
- 小程序内点击"交换名片"
- 不受手机兼容性限制

#### 方案B：纯NFC（限制多）
- 安卓→安卓：可以传数据
- 苹果限制多，很难直接传
- 需要处理各种兼容问题

#### 本方案选择
采用**方案A**，更稳定，兼容性好。

##### 实现细节
- NFC标签写入小程序URL
- 用户碰一下打开小程序
- 小程序内"交换名片"按钮
- 后端记录交换历史

## 3. 展示模块自定义

### 3.1 问题
不是每个人都一样：
- 程序员：工作年限、技术栈
- 设计师：作品集数量
- 老板：可能不需要这些

### 3.2 解决方案

#### 模板制
提供几种模板：
1. **程序员模板**
   - 工作年限
   - 技术栈
   - 项目经验
   
2. **设计师模板**
   - 作品集链接
   - 擅长风格
   - 从业年限

3. **企业老板模板**
   - 公司名称
   - 业务范围
   - 合作方式

4. **通用模板**
   - 简介
   - 联系方式

#### 编辑时选择
- 首次编辑选模板
- 之后可以切换
- 字段可微调
