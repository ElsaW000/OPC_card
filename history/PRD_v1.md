# OPC 名片小程序 - 产品规划

## 一、产品定位

帮助 AI 时代的新个体，更轻、更快地介绍自己、建立连接、沉淀联系人。

**它不是传统销售型商务名片，而是更适合展示"我是谁、我在做什么、我能和你如何连接"的个人连接工具。**

## 二、目标用户

面向 AI 时代快速涌现的新个体与轻团队，包括但不限于：

- OPC / 一人公司
- 独立开发者
- 小公司或微型团队创始人
- 独立游戏制作人
- 漫剧作者 / 创作者
- 设计师 / 插画师 / 数字内容创作者
- 项目制自由职业者

### 用户特征

- 身份复合，不止一个职业标签
- 经常需要快速介绍自己
- 既有线下连接，也有微信内远程连接
- 需要展示作品、项目、方向，而不只是留一个电话
- 社交后容易忘记对方是谁，需要后续沉淀和备注

## 三、字段设计

相比传统名片的"公司-部门-座机"，更适合的是：

- 身份标签
- 一句话介绍
- 当前项目
- 作品链接
- 社媒入口
- 微信方式

## 四、AI 功能

详见 [AI功能设计PRD.md](./AI功能设计PRD.md)

### Phase 2 - 社交公关秘书

**悬浮球设计**
- 位置：右侧悬浮
- 状态：展开/收起（不用时缩小成球）

**开头 prompt 示例**
- "您有什么需求？我这边可以帮忙找人。"
- "最近您的星标联系人【某某】更新了【某某产品/视频】，您要看看吗？"
- "请问最近有什么新的业务联系需求吗？或者问他最近生活事业还好吗？"

- [ ] AI 悬浮球（右侧悬浮，可展开/收起）
- [ ] 来访者分析
- [ ] 联系人推荐（建议可以联系谁）
- [ ] 联系反馈系统（交换后询问联系情况）
- [ ] 备注记录（记录联系结果：成功/失败/待跟进）
- [ ] 话术建议（联系人时可以说...）
- [ ] 兴趣匹配

## 四、页面结构

### 4.1 完整页面清单

| 页面 | 路径 | 说明 |
|------|------|------|
| 我的名片 | pages/mycards/mycards | 名片管理列表 |
| 联系人 | pages/contacts/contacts | 联系人列表 |
| 工作台 | pages/home/home | Dashboard |
| 管理 | pages/management/management | 设置 |
| 交换名片 | pages/exchange/exchange | 交换流程 |
| 编辑名片 | pages/edit/edit | 编辑页面 |
| 名片详情 | pages/cardDetail/cardDetail | 名片展示 |
| 访客记录 | pages/visitor/visitor | 谁看了名片 |
| 数据分析 | pages/analytics/analytics | 数据统计 |
| 会员权益 | pages/member/member | PRO会员 |

### 4.2 TabBar 结构
- 左侧3个Tab：我的名片、联系人、工作台
- 中间：交换名片按钮（突出）
- 右侧：管理

### 4.3 核心页面跳转流程

```
我的名片 ←→ 交换 ←→ 联系人 ←→ 工作台
    ↓           ↓         ↓
  编辑      访客记录    数据分析
    ↓
  名片详情
```
